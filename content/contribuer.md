Les contributions passent par GitHub pour garder une trace claire des propositions et des discussions.

## Proposer une nouvelle entrée

Ouvrir une issue de type "Nouvelle entrée" en indiquant le terme, son intérêt pour le CSCW et quelques références de départ.

## Signaler une correction ou une discussion

Ouvrir une issue de type "Correction / débat" pour signaler un problème, un angle mort, une formulation contestable ou une référence manquante.

## Soumettre une modification directe

Créer une branche, modifier ou ajouter un fichier dans `content/entries/`, puis ouvrir une pull request.

Les liens GitHub sont configurables dans `hugo.toml`.
